# 測試與問答

> 部署完跟著「上線前驗收」跑一遍。上線後遇到問題對照「FAQ」與「故障排查」。
> 部署本身的問題請看 [`DEPLOYMENT.md`](DEPLOYMENT.md) 的「常見錯誤排查」。

## 一、上線前驗收清單

部署完成後,**跟著這份清單一項一項勾**,全綠才能上線給員工用。

### A. 後端基本檢查(在 Apps Script 編輯器內)

- [ ] 執行 `verifySetup` → 4 個分頁全部 ✅,兩個金鑰都「已設定」
- [ ] 執行 `testGetCategories` → 看到你建的類別清單
- [ ] 執行 `testGetEmployees` → 看到你建的員工清單,每位 `active: true`
- [ ] 執行 `testClaude` → 回傳一個 JSON,裡面有 `category` 與 `keyword`
- [ ] 執行 `testStats` → 回傳今天 0 筆(若還沒測過寫入)

### B. 主管端(admin.html)檢查

開 admin.html,確認:

- [ ] 頁面正常載入,沒有「⚠️ 請先設定 API_URL」紅色提示
- [ ] 頂部時間戳記 + 筆數會顯示
- [ ] 切到「類別管理」分頁,看到你建的類別
- [ ] 切到「員工管理」分頁,看到員工,啟用切換可以開關
- [ ] 試新增一個假類別 `TEST`,儲存,再刪掉
- [ ] 試新增一個假員工 `測試員`,啟用切換切看看,**保留它待會員工端要用**
- [ ] 篩選列的「操作員」「類別」下拉都有選項

### C. 員工端(mobile.html)拍照流程

員工手機開 mobile.html(必須是 HTTPS,即 GitHub Pages 網址),確認:

- [ ] 頁面載入沒有錯誤提示
- [ ] 「操作員」下拉看到「測試員」
- [ ] 選了「測試員」之後,**重新整理頁面**仍記得選擇(localStorage 測試)
- [ ] 按「📷 拍照」 → 第一次會跳「允許相機」彈窗,允許後相機開啟
- [ ] 拍下任意商品照片(玩具、衣服都可以) → 自動進入「判斷中」 → 5–15 秒後出現大字結果
- [ ] 結果區整片底色變色 + 大字顯示「丟到 X 區」
- [ ] **手機震一下**(若手機有震動功能)
- [ ] 包裹編號輸入框自動 focus(綠框 + 閃)

### D. 員工端文字輸入流程

- [ ] 在文字框打 `兒童玩具` → 按「✓ 判斷文字」 → 出現分類結果
- [ ] **再次**打 `兒童玩具` → 結果出現得很快(< 1 秒),且來源欄顯示「快取」
  - 這代表自動學習與快取攔截都正常運作

### E. 包裹編號送出

- [ ] 在包裹編號框打 `TEST-001` → 按 Enter 或「送出」 → 看到「✅ 已記錄」
- [ ] 畫面自動清空,回到等下一個包裹的狀態
- [ ] 切到 admin.html → 10 秒內看到剛才那筆紀錄
- [ ] 切到 Google Sheet 「包裹紀錄」分頁 → 最後一列是新增的紀錄,**操作員欄位是「測試員」**

### F. 行內修正分類

- [ ] 在 admin.html 「包裹紀錄」分頁,點剛才那筆的「類別」欄位 → 變成下拉
- [ ] 改成另一個類別 → 看到 ✅ 提示
- [ ] 該筆的「來源」欄變成「人工」
- [ ] 切到 Sheet 「關鍵字快取」分頁 → 觀察被改掉的關鍵字,類別應該也跟著變

### G. CSV 匯出

- [ ] 在 admin.html 按「📥 匯出 CSV」 → 瀏覽器下載 `包裹紀錄_YYYY-MM-DD.csv`
- [ ] 用 Excel 開啟 → 中文不亂碼,欄位齊全

### H. 條碼槍(若有)

- [ ] 條碼槍配對手機完成(設定 → 藍牙看得到 connected)
- [ ] 開 mobile.html,先選操作員,先拍一張照走完判斷
- [ ] 包裹輸入框 focus 著 → 用條碼槍掃任何條碼 → 自動填入 + 自動送出
- [ ] 看到「✅ 已記錄」

### I. QR 掃描(備援)

- [ ] 在包裹編號 focus 狀態下,按 📷 → 相機開,黑底覆蓋層出現
- [ ] 對著任意 QR 掃 → 自動填入 + 自動送出

### J. 清理測試資料

- [ ] 在 Google Sheet「包裹紀錄」分頁,把剛才測試的列(包裹號 `TEST-001` 那筆)刪掉
- [ ] 在 admin.html「員工管理」把「測試員」停用(或在 Sheet 直接刪那列)

## 二、上線後日常檢查

### 每天

- 主管隨手開 admin.html,看「來源」欄是否大量出現「快取」(代表省錢中)
- 看「失敗」筆數是不是異常多(超過 5 筆/天就要查 Anthropic console)

### 每週

- 進「關鍵字快取」分頁,觀察:
  - 命中次數高的關鍵字(代表學得很好)
  - 看起來怪的關鍵字 → 主管手動刪除避免誤判
- console.anthropic.com → Usage 看週用量,推估月底會花多少

### 每月

- 開帳單看實際支出,跟預估的 NT$100–900 對照
- 把當月 CSV 匯出存檔,作為紙本/合約存底

## 三、員工教學重點

> 把這幾條印出來貼倉庫牆上。

### 第一次用

1. 開員工專屬網址(主管會給)
2. **頂部下拉選自己的名字**(只要選一次,以後自動記住)
3. 第一次按「📷 拍照」會跳允許相機 → **點允許**

### 每個包裹的標準動作

```
拿起包裹
  ↓
找到外箱上印有商品描述的標籤
  ↓
按「📷 拍照」對準標籤拍下去(整張清楚就好)
  ↓
螢幕大字告訴你丟到哪區 → 把包裹搬過去
  ↓
條碼槍掃包裹條碼,聽到嗶 → 看螢幕「✅ 已記錄」
  ↓
下一個!
```

### 遇到狀況

- **拍模糊了**:再按一次「📷 拍照」會重拍,舊的會被取代
- **AI 判斷錯了**:照原本說的丟,主管會在後台修正(系統會自動學)
- **AI 說「未分類」**(紅底警示):放到「待處理區」,主管會手動分類
- **網路斷了**:不會送出,會看到紅色錯誤;網路恢復後重拍即可
- **不知道丟哪**:詢問主管,**不要亂猜**,亂猜會教壞 AI

### 不要做的事

- ❌ 不要拿別人的名字操作(紀錄會錯)
- ❌ 不要按到「📥 匯出」按鈕(那是主管端,員工端沒有)
- ❌ 不要把員工網址傳給沒在我們倉庫工作的人

## 四、FAQ

### 使用面

**Q1. AI 一直判斷錯怎麼辦?**

A. 有三種情況:
1. **整體不準**(各種商品都錯) → 通常是「類別清單」太籠統。把類別細分,例如把「服飾」拆成「男裝/女裝/童裝」,AI 會更好判斷
2. **某幾類老錯**(例如所有食品都被判成日用品) → 主管在 admin.html 行內改正,系統會反向更新快取
3. **照片模糊** → 教員工拍清楚一點,或直接打字描述

**Q2. 同一商品判斷結果不一致?**

A. 第一次走 AI(可能有微小差異),第二次起應該走快取(穩定)。如果第二次還在走 AI,代表關鍵字快取沒命中 → 檢查「關鍵字快取」分頁有沒有東西。

**Q3. 我想完全相信員工的判斷,不靠 AI。**

A. 這個系統就用不到了 😄。但如果你只是想覆蓋 AI:在 admin.html 行內改類別即可,系統會記住。

**Q4. 可以一次掃多個包裹批次處理嗎?**

A. 目前一次一個。批次模式比較複雜(要判斷哪張照片配哪個包裹號),先把現在這個跑順再擴充。

### 效能與成本

**Q5. 一個 Sheet 最多寫多少筆會撐不住?**

A. Google Sheet 單檔上限 **1000 萬個儲存格**。「包裹紀錄」分頁 7 欄 → 約 140 萬筆。每天 300 包要寫 12 年才會滿。但建議每年初手動把上一年的紀錄另存一個 Sheet,讓主表保持輕量、admin 載入快。

**Q6. 多人同時掃會衝突嗎?**

A. 不會。Google Sheet 的 `appendRow` 是 atomic 的,Apps Script 也會排隊處理。極端情況可能有人等 1–2 秒,不會丟資料。

**Q7. AI 太貴怎麼辦?**

A. 三招:
1. 細看「來源」欄,正常 80%+ 應該是「快取」。如果都是「Claude(圖)」,代表員工每次描述都不一樣 → 教員工拍同一個位置的標籤
2. 重複的商品文字描述會比拍照便宜很多。鼓勵員工先用文字描述試一次(對固定回頭客的商品很有效)
3. 在 admin.html 「關鍵字快取」分頁手動加常見關鍵字 → 命中率會大幅提高

**Q8. Claude 用量限制(rate limit)會撞到嗎?**

A. Anthropic Tier 1 限制 50 RPM(每分鐘 50 次)。倉庫每分鐘最多撈 12 包(5 秒一包),離上限很遠。除非每分鐘超過 50 包才需要升 Tier(自動,看 console.anthropic.com)。

### 資料

**Q9. 我想把資料搬到其他地方(例如自己的資料庫)?**

A. 隨時可以。打開 Google Sheet 直接「檔案 → 下載 → CSV / Excel」就拿到所有資料。或在 admin.html 用 CSV 匯出。

**Q10. Sheet 不小心刪了一筆怎辦?**

A. Google Sheet 有版本歷史:**檔案 → 版本記錄 → 查看版本記錄**,可以還原到任何時間點。

**Q11. 我想關掉這個系統一陣子(例如過年休倉)?**

A. 不用關。沒有員工開 mobile.html 就不會打 API,不會花錢。Anthropic 帳戶不收月費。

### 安全

**Q12. 員工把 mobile.html 網址外傳怎辦?**

A. 影響有限,因為:
1. 沒有員工名字選不到操作員,送不出資料
2. 但外人能看到你的類別清單與員工清單

如果擔心:把 GitHub repo 設成 Private(GitHub Pro 才有 Pages)。或把 mobile.html / admin.html 名稱改成隨機字串(例如 `mobile-x9k2f8.html`)讓網址不好猜。

**Q13. API Key 會外洩嗎?**

A. 不會。Key 只存在 Apps Script 的 `PropertiesService`,前端從來看不到。Apps Script 程式碼本身就算公開也沒事(沒寫 Key)。

**Q14. 主管網址(admin.html)被員工發現怎辦?**

A. 員工最多就是看到歷史紀錄、改類別、加員工。不能刪資料(只有 Sheet 擁有者能)。
若擔心,改 admin.html 檔名為隨機字串。

## 五、故障排查決策樹

```
員工說「壞了」
  │
  ├─ 完全打不開頁面
  │    └─ 檢查 GitHub Pages 是否還是 enabled(Settings → Pages)
  │       檢查公司網路是否擋 github.io
  │
  ├─ 頁面開了但下拉是空的
  │    └─ Web App URL 換了沒填回 HTML?(改了 Code.gs 重部署 URL 不變,不應該換)
  │       或員工/類別 Sheet 被誤改
  │
  ├─ 拍照後無限轉圈
  │    ├─ 開瀏覽器 console(電腦版用 F12)看錯誤訊息
  │    ├─ Anthropic console 看是否儲值用完
  │    └─ Apps Script 「執行記錄」看是否有錯
  │
  ├─ 結果出來但「來源」是失敗
  │    └─ Apps Script「執行記錄」看 Claude 回什麼錯誤
  │
  ├─ 條碼槍不動
  │    ├─ 藍牙連線斷掉(設定 → 藍牙重連)
  │    ├─ 手機輸入法切到中文了(切回英文)
  │    └─ 焦點不在輸入框(點一下框)
  │
  └─ 資料沒寫進 Sheet
       └─ Apps Script 帳號和 Sheet 帳號不同(設定權限)
          檢查 Sheet ID 是否正確
```
